# 🪖 坦克大战网页版

一个基于 **HTML5 Canvas** 的硬核复刻项目。本项目不仅还原了经典《坦克大战》的逻辑，更深度集成了变态 AI 与道具反制机制，并采用现代前端工程化标准构建。

## 🌟 核心工程亮点

* **🚫 零资源依赖 (Zero-Asset Architecture)**: 所有视觉元素（坦克、地形、特效）均采用程序化绘图（Canvas API）渲染，甚至音效也由 **Web Audio API** 实时合成。
* **🛠 确定性碰撞系统**: 针对 $20 \times 20$ 像素网格实现的 AABB 矩形碰撞检测，确保在 60FPS 下逻辑的一致性。
* **🧠 AI 逻辑**: 敌方坦克具备“贪婪路径倾向”，会主动抢夺地图道具并对玩家造成降级打击。
* **🔊 音效合成引擎**: 摒弃传统的 MP3/WAV，使用振荡器（Oscillator）模拟 8-bit 处理器的方波和噪声效果。

## 🏗 技术架构 (System Architecture)

项目采用解耦的**系统驱动设计**，方便进行物理参数调节与逻辑测试：

* **GameLoop**: 基于 `requestAnimationFrame` 的插值驱动，确保不同刷新率屏幕下的运动匀速。
* **CollisionSystem**: 采用分层检测策略，优先检测静态网格（砖/钢），再进行动态实体（坦克/子弹）检测。
* **State Machine**: 严格管理 `HOME` -> `LEVEL_START` -> `BATTLE` -> `SCORE` -> `GAMEOVER` 的状态转移。

## ⚙️ 核心逻辑规格

### 1. 物理坐标系

游戏内部逻辑运行在战场坐标系中：

* **战场总尺寸**: $600 \times 760$ 单位。
* **基础单元格 (Tile)**: $20 \times 20$ 单位。
* **坦克包围盒 (AABB)**: $40 \times 40$ 单位。
* **碰撞容错**: $\pm 2$ 像素，用于优化转向时的“卡墙”体验。

## 🛠️ 开发者手册

### 开发环境启动

```bash
npm install
npm run dev   # 极速 Vite 热更新环境

```

### 自动化测试

针对核心物理引擎和碰撞算法，项目配置了完善的单元测试：

```bash
npm run test  # 验证碰撞边界逻辑与状态机转移

```

### 地图扩展规范

地图存储在 `src/world/levels/levelXX.txt`。每行 30 个整数，通过简单的矩阵映射即可自定义关卡。
